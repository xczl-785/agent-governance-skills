# Roadmap Split Example

## Situation

A roadmap file contains:

- current goals
- completed release summaries
- version-by-version recap blocks
- temporary execution notes

## Wrong Response

Keep appending into the same roadmap because "people already know where it is."

This preserves location familiarity while destroying reading role clarity.

## Better Response

- keep the roadmap focused on current and forward-looking planning
- move completed-version summaries into recap or history material
- reduce exposure of old completion blocks on the active path

## Why This Matters

In `agent-first` use, an overloaded roadmap stops being a planning entry and becomes a mixed-responsibility trap.
